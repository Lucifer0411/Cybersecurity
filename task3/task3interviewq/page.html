1.What is vulnerability scanning?

Vulnerability scanning is the process of automatically identifying security weaknesses in systems, networks, or applications. It uses specialized tools to scan for known vulnerabilities, such as outdated software, misconfigurations, or missing patches.

The main goal is to detect and prioritize potential security risks before attackers can exploit them. Common tools include Nessus, OpenVAS, and Qualys.

Example:“As a cybersecurity analyst, I schedule regular vulnerability scans to identify system weaknesses and then work with the IT team to remediate or patch them before they become threats.”

2.Difference between vulnerability scanning and penetration testing?

The main difference lies in depth and intent:

Vulnerability Scanning is an automated process that identifies known weaknesses in systems or applications. It tells you whatvulnerabilities exist but doesn’t exploit them. (Example: Using Nessus or OpenVAS to find outdated software or weak configurations.)


Penetration Testing is a manual, in-depth assessment performed by ethical hackers to actively exploit vulnerabilities and determine the real impact of an attack. (Example: Using Burp Suite or Metasploit to test if a vulnerability can lead to data breach.)
Vulnerability Scan → Finds flaws
Penetration Test → Exploits flaws to measure risk

3.What are some common vulnerabilities in personal computers?

Common vulnerabilities in personal computers include:


Unpatched Software:Missing updates in the OS or applications can allow attackers to exploit known bugs.


Weak Passwords:Simple or reused passwords make it easy for brute-force or credential stuffing attacks.


Malware Infections:Downloading malicious files or clicking phishing links can install viruses, trojans, or ransomware.


Unsecured Network Connections:Using public Wi-Fi without encryption can lead to data interception.


Misconfigured Security Settings:Disabled firewalls, outdated antivirus, or lack of system hardening.


Removable Media Risks:Using infected USB drives can introduce malware.


Example answer:
“As a cybersecurity analyst, I ensure regular patching, enforce strong password policies, and educate users about phishing and safe browsing to minimize these vulnerabilities.”

4.How do scanners detect vulnerabilities?

Vulnerability scanners detect vulnerabilities by comparing system information against a database of known security issues. They analyze configurations, software versions, and network responses to identify weaknesses.


Here’s how they generally work:
  • Asset Discovery:Identify devices, ports, and services running on the network.
  • Fingerprinting:Determine OS and software versions.
  • Database Comparison:Match findings against a database of known CVEs (Common Vulnerabilities and Exposures)

Verification:Run safe tests or simulated exploits to confirm vulnerabilities.


Reporting:Generate a risk-based report for remediation.

Example:
“Tools like Nessus or Qualys use CVE databases to detect outdated or misconfigured software, then score risks using CVSS to help prioritize fixes.”

5.What is CVSS?

CVSS (Common Vulnerability Scoring System)is a standardized framework used to measure the severity of security vulnerabilities. It assigns a score from 0 to 10, where higher values indicate more critical risks.


It’s based on three metric groups:
  • Base Metrics:Fundamental characteristics (e.g., exploitability, impact).
  • Temporal Metrics:How the threat changes over time.
  • Environmental Metrics:How it affects a specific organization.

Example:
“A vulnerability with a CVSS score of 9.8 is considered Critical, meaning it requires immediate remediation to prevent exploitation.”

6.How often should vulnerability scans be performed?

Vulnerability scans should be performed regularly and strategicallyto ensure continuous security.


Here’s the general guideline:
  • Routine Scans:At least monthly or quarterlyfor standard systems.
  • Critical Systems:Weekly or bi-weekly, depending on risk level.
  • After Major Changes:Whenever new software, patches, or configurations are applied.
  • After Security Incidents:To verify that no new vulnerabilities exist.

Example:
“As a cybersecurity analyst, I schedule weekly scans for critical servers and monthly scans for the overall network, ensuring vulnerabilities are identified and patched promptly.”

7.What is a false positive in vulnerability scanning?


A false positivein vulnerability scanning occurs when the scanner reports a vulnerability that doesn’t actually existon the system.

This usually happens due to outdated vulnerability databases, network timeouts, or misconfigured scan settings. While the scanner flags it as a risk, manual verification later proves it safe.

Example:
“In my previous scans, I’ve seen cases where a tool like Nessus marked a service as vulnerable because of a version mismatch, but after manual testing, we confirmed it was already patched — a classic false positive.”

8.How do you prioritize vulnerabilities?

I prioritize vulnerabilities based on risk and impactto ensure critical issues are addressed first. The key factors I consider are:


CVSS Score:Higher scores (7.0–10.0) are treated as urgent.

Exploit Availability:If public exploits exist, it becomes top priority.

Asset Criticality:Systems handling sensitive data or core operations get prioritized.

Exposure Level:Internet-facing or publicly accessible systems are more urgent.

Business Impact:How much downtime or data loss the vulnerability could cause.

Example:
“I use a risk-based approach — focusing first on critical and exploitable vulnerabilities on high-value assets, then moving to medium and low-severity issues.”
